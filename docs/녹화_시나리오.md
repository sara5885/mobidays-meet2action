# 화면 녹화 시나리오 (1~3분)

> 제출용 데모 영상 대본. Loom / QuickTime(맥 기본) / unlisted YouTube 중 택1.
> 목표: "음성/transcript → 액션아이템 → 의사결정 대시보드"가 한 흐름으로 돈다는 걸 보여주기.
> 길이 목표 2분 30초. 말은 천천히, 화면 전환은 또렷하게.

## 사전 준비 (녹화 시작 전)
- 터미널 1개 + 브라우저(대시보드) 준비. 글자 크기 키우기(터미널 14pt+).
- `make clean` 으로 DB 비워두기 (라이브로 적재되는 걸 보여주기 위해).
- `.env` 의 키 줄은 화면에 비추지 말 것(보안). `cat .env` 금지.
- (선택) `make gemini-check` 한 번 미리 돌려 quota 워밍업.

## 타임라인

### [0:00–0:20] 도입 — 무엇을 만들었나
- **화면**: README 상단 또는 기획안 1페이지.
- **멘트**: "모비데이즈 회의는 정리에 30~60분이 들고, 한국어 회의 특성상 액션아이템 누락이
  잦습니다. 이걸 자동화하되, 신뢰할 수 있게 만드는 시스템을 구현했습니다."

### [0:20–0:45] 입력 데이터 보여주기
- **화면**: `data/raw/ko_meeting_3speakers.json` 잠깐 스크롤 — 화자·발화 구조.
- **멘트**: "제공된 실제 회의 transcript입니다. 머뭇거림, '그건 제가 챙길게요' 같은 암묵적 R&R,
  '일단 두고 봐요' 같은 흐릿한 결정이 그대로 들어있습니다."

### [0:45–1:15] 파이프라인 실행 (라이브)
- **화면**: 터미널에서 `make demo` 실행. 로그가 흐르는 걸 보여줌.
- **멘트**: "한 줄 실행이면 정제 → 청크 → DuckDB 적재 → **실제 Gemini 추출**까지 돕니다.
  로그에 provider=gemini, 회의별 액션아이템 수, 그리고 약어 용어집이 자동 인식된 게 보입니다.
  멱등성을 보장해서 재실행해도 중복이 쌓이지 않습니다."
- **팁**: 로그에서 `provider=gemini` 와 `action_items=N` 부분에 마우스로 가리키기.

### [1:15–2:10] 대시보드 — 의사결정 흐름
- **화면**: `make dashboard` → 브라우저. 위젯을 위에서 아래로 짚으며.
- **멘트** (위젯마다 한 문장씩):
  - "① 주차별 추이 — 막대는 발생 건수, 선은 완료율. 워크로드가 어느 주에 몰리는지 보입니다."
  - "② 담당자별 미완료 Top N — 누가 병목인지, 업무 재분배 판단."
  - "③ 반복 이슈 키워드 — '전환', '픽셀'이 여러 회의에 반복 등장. 일회성이 아니라
    구조적 과제라는 신호입니다."
  - "④ confidence 분포 — **신뢰도 낮은 항목만 사람이 검수**합니다. 아래 드릴다운에
    '그건 제가 챙길게요'처럼 책임자가 흐릿한 발화가 낮은 confidence로 잡혀 있습니다."

### [2:10–2:30] 신뢰성·상태관리 마무리
- **화면**: 하단 "✏️ 액션아이템 상태 관리"에서 한 건을 '완료'로 바꾸고 저장 → 위젯 갱신.
- **멘트**: "담당자가 진행상황을 갱신하면 추이·미완료 위젯에 반영됩니다. 이 상태는 별도
  레이어에 저장돼서 **파이프라인을 다시 돌려도 사람이 바꾼 값이 보존**됩니다.
  핵심은, 신뢰할 수 없는 LLM 출력을 검증·재시도·confidence로 신뢰 가능한 데이터로 만든 것입니다."
