# AI 활용 내역 (AI_USAGE.md)

본 과제는 AI 도구를 적극 활용하되, **결과물을 비판적으로 판단·수정**하는 데 초점을 뒀습니다.
"AI가 만든 초안"을 그대로 쓰지 않고, 실제로 돌려보며 발견한 문제를 직접 고친 사례를 중심으로 정리합니다.

## 사용한 AI 도구와 이유
- **Claude (Cowork)** — 아키텍처 설계 토론, 코드 스캐폴딩, 프롬프트 초안, 디버깅 동반, 문서 작성.
  대화형으로 설계 의사결정을 함께 점검하는 용도.
- **Gemini 2.5 Flash** — 액션아이템 추출 LLM 중 하나 (`LLM_PROVIDER=gemini`). 무료 + native JSON.
- **Ollama (qwen2.5:7b, 로컬)** — 무료·무제한·온프레미스 LLM (`LLM_PROVIDER=ollama`).
  외부 유출 금지 제약을 실제로 충족하기 위해 도입.
- **Whisper (로컬)** — 음성 → 텍스트 STT (가산점).

## 의사결정 과정 / 컨텍스트
- **walking skeleton 전략**: 시간 제약 하에 end-to-end 한 줄기를 mock으로 먼저 완성하고,
  배점 큰 순서(LLM 신뢰성 → 정제 → 대시보드 → 가산점)로 두껍게 확장.
- **provider 추상화**: mock / gemini / ollama 를 환경변수 하나로 토글. 외부 유출 금지 제약,
  rate limit, 시연 안정성을 동시에 만족시키기 위한 설계.
- **추출/트래킹 레이어 분리**: 멱등성 요구와 진행상황 업데이트를 한 설계로 해결.
- **컨텍스트 파일**: `docs/기획안.md`(설계 근거), `docs/코드_가이드.md`(파일별 해설)를 두어
  AI와의 협업·재개 시 맥락이 유지되도록 함.

## AI 결과물을 직접 수정/판단한 사례
실제 실행 중 발견해 고친 것들 (무엇을 / 왜):

1. **eval로 타입 변형 발견 → '거부'에서 '보정'으로 전환** *(가장 중요)*
   precision/recall 평가(`eval_extraction.py`)를 돌렸더니 한 회의가 0건으로 나옴.
   원인: 소형 로컬 모델이 `source_quote`를 문자열 대신 **리스트로** 반환 → pydantic이 거부 → 전체 손실.
   → 검증기를 "형식 틀리면 거부"에서 "**리스트면 합쳐 문자열로 보정**"으로 바꿈(`schemas.py`).
   결과: **F1 0.67 → 0.81**. AI 초안의 엄격한 검증을 실제 데이터로 완화 판단한 사례.

2. **confidence 후처리 추가** — AI 초안은 LLM이 준 confidence를 그대로 신뢰. 담당자 불명확하거나
   인용 근거가 입력에 없으면(환각) confidence 상한을 거는 `_postprocess`를 직접 추가.

3. **추출/트래킹 레이어 분리 재설계** — 초기엔 status를 action_items에 두었으나, 재실행 시
   사람이 바꾼 상태가 사라지는 문제를 발견 → action_status/status_history로 분리 + action_key 연결.

4. **빈 액션아이템 적재 크래시 + 429 백오프** — Gemini 무료 티어 분당 5회 제한으로 한 회의가
   0건이 됐을 때 DuckDB `executemany`가 죽는 버그를 실행 중 발견 → 빈 리스트 가드 + 429 대기·재시도 추가.

5. **로더 방어적 파싱** — 제공 transcript의 실제 구조가 AI 가정과 달라(speaker=이름, role 내장,
   타임스탬프 없음) 두 포맷을 모두 흡수하도록 수정.

6. **BoW 키워드 잡음 제거** — 초기 키워드에 "제가/내가/채린님이" 같은 대명사·호칭이 섞여,
   조사 제거 + 불용어 + 화자 이름 필터를 직접 보강.

7. **STT 스크립트 일반화** — AI 초안은 화자(지훈/수아/채린)와 파일 경로가 하드코딩 → 임의 mp3를
   받고 화자를 LLM이 추론하도록 일반화.

8. **diarization 출력 설계 변경(타임아웃 해결)** — 처음엔 LLM이 42개 발화 텍스트를 통째로 다시
   출력하게 했더니 로컬 모델에서 타임아웃. → "id→화자" **매핑만** 받고 텍스트는 원문을 붙이도록
   변경 → 출력 1/10, 속도↑, 원문 훼손 0. (실행해보고 발견한 설계 결함을 수정한 사례)

9. **참석자 명단을 추출 프롬프트에 주입** — owner를 자유 생성하던 것을 "참석자 명단(닫힌 후보)에서만
   고르게" 제약 → 실측 담당자 정확도 0.77 → 0.875로 개선(eval로 확인).

10. **직원 마스터 정규화 + is_adhoc** — meeting_participants에 이름·역할을 복제하던(비정규화) 것을
    employees(마스터)+FK로 정규화. 자동 생성되는 미상 화자가 마스터를 오염시키는 문제를 보고
    `is_adhoc` 플래그로 정식 직원과 분리.

11. **회의록 정리 추가 + 대시보드 UX** — 액션아이템만으론 "추출이 맞나" 검증이 어려워, 요약·안건·
    결정사항을 함께 뽑고 대시보드를 목록→상세 클릭 구조로 바꿔 원문과 대조 가능하게 함.

## 프롬프트 예시
- "한국어 광고 회의의 암묵적 R&R 표현('그건 제가 챙길게요')을 담당자로 매핑하는 few-shot 프롬프트를 설계해줘"
- "LLM JSON 출력을 pydantic으로 검증하고 실패 시 강화 프롬프트로 재시도하는 패턴을 짜줘"
- "재실행 시 사람이 바꾼 상태는 보존하면서 추출 결과는 멱등하게 교체하는 스키마를 설계해줘"
- "gold(수기 정리) 대비 LLM 추출의 precision/recall을 측정하는 스크립트를 만들어줘"

## 한 줄 회고
AI는 **초안을 빠르게 만드는 데** 큰 역할을 했지만, 품질을 끌어올린 결정적 지점들
(타입 보정, 레이어 분리, 환각 필터)은 **실제로 돌려보고 비판적으로 수정**한 데서 나왔습니다.
